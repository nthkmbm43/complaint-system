## 🌟 ฟีเจอร์หลัก (Key Features)

ระบบถูกออกแบบมาให้รองรับการทำงานของ 3 กลุ่มผู้ใช้งานหลัก ได้แก่ **นักศึกษา**, **เจ้าหน้าที่/อาจารย์**, และ **ผู้ดูแลระบบ (Admin)** โดยมีฟีเจอร์ที่โดดเด่นดังนี้:

### 👨‍🎓 สำหรับนักศึกษา (Students)
- **ส่งข้อร้องเรียน:** สามารถกรอกรายละเอียด หมวดหมู่ และแนบไฟล์ภาพประกอบการร้องเรียนได้
- **ติดตามสถานะ (Tracking):** ติดตามความคืบหน้าของข้อร้องเรียนได้แบบ Real-time (รับเรื่อง -> กำลังดำเนินการ -> เสร็จสิ้น)
- **ระบบแจ้งเตือน (Notifications):** มีกระดิ่งแจ้งเตือนเมื่อมีการอัปเดตสถานะข้อร้องเรียน หรือข้อความจากเจ้าหน้าที่
- **ประเมินความพึงพอใจ:** เมื่อปัญหาถูกแก้ไขเสร็จสิ้น นักศึกษาสามารถให้คะแนนความพึงพอใจต่อการปฏิบัติงานได้

### 👨‍🏫 สำหรับเจ้าหน้าที่และผู้ดูแลระบบ (Staff & Admin)
- **Dashboard & Reports:** หน้าแดชบอร์ดสรุปสถิติข้อร้องเรียนทั้งหมด แบ่งตามสถานะ หมวดหมู่ และความเร่งด่วน พร้อมระบบออกรายงาน
- **ระบบมอบหมายงาน:** สามารถมอบหมายงานให้กับเจ้าหน้าที่ที่เกี่ยวข้องโดยตรงได้
- **จัดการข้อมูลพื้นฐาน:** ผู้ดูแลระบบสามารถเพิ่ม/ลบ/แก้ไข ข้อมูลนักศึกษา ข้อมูลอาจารย์ ประเภทข้อร้องเรียน และข้อมูลหน่วยงานได้
- **ระบบรักษาความปลอดภัย:** อัปเกรดการเข้ารหัสรหัสผ่านด้วย `password_hash()` (Bcrypt) ขั้นสูง พร้อมระบบรองรับผู้ใช้เก่า (Fallback)

### 🎨 การออกแบบ (UI/UX Design)
- **Modern UI:** ออกแบบหน้าตาให้สวยงามทันสมัยด้วยเทคนิค **Glassmorphism** และ **Gradients**
- **Responsive Design:** รองรับการใช้งานผ่านโทรศัพท์มือถือ แท็บเล็ต และคอมพิวเตอร์อย่างสมบูรณ์ 100%
- **Micro-animations:** เพิ่มลูกเล่นและ Transition ในการกดปุ่มหรือเปิดเมนูเพื่อประสบการณ์ใช้งานที่ไหลลื่น

## 🛠️ โครงสร้างเทคโนโลยี (Tech Stack)
- **Frontend:** HTML5, Vanilla CSS3 (ไม่พึ่งพา Framework หนักๆ เพื่อความรวดเร็ว), Vanilla JavaScript
- **Backend:** PHP 8+ (PDO & Prepared Statements เพื่อป้องกัน SQL Injection)
- **Database:** MySQL / MariaDB
- **Security:** Bcrypt Password Hashing, Session Hijacking Prevention

## ⚙️ วิธีการติดตั้ง (Installation Guide)

1. **ดาวน์โหลด Source Code**
   ทำการ Clone หรือดาวน์โหลดไฟล์ Zip ไปวางไว้ในโฟลเดอร์สำหรับรันเซิร์ฟเวอร์จำลอง เช่น `c:\xampp\htdocs\complaint`
   ```bash
   git clone https://github.com/nthkmbm43/complaint-system.git
   ```

2. **ตั้งค่าฐานข้อมูล (Database)**
   - เปิด XAMPP Control Panel และ Start บริการ **Apache** และ **MySQL**
   - เข้าไปที่ `http://localhost/phpmyadmin`
   - สร้างฐานข้อมูลใหม่ (Create Database) ชื่อว่า `complaint_db` (หรือชื่อตามไฟล์ SQL ของคุณ)
   - นำเข้า (Import) ไฟล์ฐานข้อมูล `.sql` ที่แนบมากับโปรเจกต์ลงในฐานข้อมูลที่เพิ่งสร้าง

3. **ตั้งค่าการเชื่อมต่อ (Configuration)**
   - ไปที่โฟลเดอร์ `config/`
   - เปิดไฟล์ `database.php` (หรือไฟล์ที่เกี่ยวข้องกับการเชื่อมต่อ) 
   - แก้ไขการตั้งค่าให้ตรงกับเครื่องของคุณ:
     ```php
     define('DB_HOST', 'localhost');
     define('DB_USER', 'root');
     define('DB_PASS', '');
     define('DB_NAME', 'complaint_db');
     ```

4. **ตั้งค่า Permission ของโฟลเดอร์อัปโหลด**
   - ตรวจสอบให้แน่ใจว่าโฟลเดอร์ `uploads/` หรือโฟลเดอร์ที่ใช้เก็บรูปภาพ สามารถอ่านและเขียนได้ (Read & Write Permissions)

5. **รันระบบ**
   - เปิด Web Browser และพิมพ์ URL: `http://localhost/complaint`
   - เข้าสู่ระบบด้วยบัญชีผู้ใช้เริ่มต้นที่มีในฐานข้อมูล
